Perguntas

1:Para que serve o commit e qual a melhor forma de utilizá-lo?
Commit serve para enviar e salvar os dados e código de armazenamento em um banco de dados, ele é mais usado para fazer controle de versão. no caso do controle de versão refere-se a submeter as últimas alterações do código fonte ao repositório e tornar estas alterações parte da versão principal (head) do repositório. Deste modo, quando outros usuários fazem um “UPDATE” ou um “CHECKOUT” no repositório, eles conseguem receber a versão mais atualizada enviada para eles. Existem várias maneira de se utilizar e comitar o commit, mais quando falamos de commit pensamos em alterações no repositório,essas alterações são feitas através do git que quando o arquivo novo adiciona apenas o que já foram versionados, os que não foram versionados eles continuam igual.

2:Qual a diferença entre commit e push?
O commit serve para registrar as alterações no repositório, essas alterações são feitas na máquina local, mais no caso ele não conseguiria marcas as mudanças realizadas no repositório remoto. enquanto o git push consegue atualizar as referências remotas com os objetos associados a ele, a principal diferença entre eles e que o commit tem uma conexão com o repositório local, já o push é usado como um repositório remoto.

3:O que é tag?
R: Uma tag é usada para rotular e marcar um commit específico no histórico. Geralmente é usado para marcar pontos de liberação (por exemplo, v1.0, etc.).

4:O que é branch?
R: É uma ramifcação de um desenvolvimento. É uma cópia do código derivado de um certo ponto do código principal que é utilizado para a aplicação de mudanças no código, preservando a integridade do código na ramificação principal.

5:Como resolver possíveis conflitos no Git?
R: Os conflitos surgem quando duas pessoas alteram o mesmo arquivo, isso acontece quando as duas partes alteram a mesma linha do arquivo, é preciso que o usuário resolva os conflitos um por um manualmente, ou através de uma ferramenta específica para esse fim, porém é um processo delicado e tedioso, principalmente quando o problema ocorre em vários arquivos.

6:Em caso de incêndio, quais são os comandos git necessários antes de sair do prédio?
R: Primeiro o comando → git add -A para adicionar todos os arquivos → git commit você define um ponto de verificação para poder acessá-lo posteriormente → git push para enviar e salvar alterações no repositório remoto.
